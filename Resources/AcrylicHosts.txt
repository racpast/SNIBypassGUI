# 如果您对此文件进行任何修改，则必须重新启动Acrylic DNS代理服务或控制台才能看到其效果。
#
# 这是AcrylicHosts.txt文件。
#
# 它包含了与本地Windows HOSTS文件类似的域名和地址之间的预定义映射，但有一些增强功能。
#
# 标准格式为：IP地址 域名1 [域名2] [域名3] ...
#
# 其中，IP地址使用点分十进制表示（IPv4）或使用冒号分隔的组表示（IPv6），
# 域名1、域名2和域名3为字符串。
#
# 以“#”字符开头的行（如果在一行中找到它，则在它后面的所有内容）被视为注释，
# 因此会被忽略。
#
# 域名可以包含通配符字符“*”（匹配零个或多个字符）和“?”（匹配一个字符）：
#
# 127.0.0.1 ad.* ads.*
#
# 域名也可以是正则表达式，如果以“/”字符开头：
#
# 127.0.0.1 /^ads?\..*$/
#
# 请注意，正则表达式末尾没有“/”。关于正则表达式引擎及其语法的更多信息，
# 请访问：
#
# http://www.pcre.org/
#
# 以“>”字符开头的域名是一个方便的快捷方式，用于表示所有以该字符后缀的域名。
# 例如，以下条目：
#
# 127.0.0.1 >google.com
#
# 等效于：
#
# 127.0.0.1 google.com *.google.com
#
# NXDOMAIN（或负响应）和FORWARD条目也可以映射到域名，使用上面解释的所有可能性
# （即通配符字符和正则表达式）：
#
# NX www.google.com
# NX >google.com
# NX /^ads?\..*$/
#
# FORWARD条目总是首先进行评估，并可用于指定其他规则的例外。例如，以下配置：
#
# FW >apple.com
# FW >google.com
# FW >microsoft.com
# NX *
#
# 指示Acrylic将“apple.com”、“google.com”和“microsoft.com”结尾的域名请求转发到您的
# DNS 服务器，并且对所有其他域名返回NXDOMAIN响应，从而有效地阻止除了您显式允许的域名以外的所有域名。
#
# 为了更方便地维护来自多个来源的条目，也可以使用以下语法“包含”外部HOSTS文件
# （该行必须以“@”字符开头，后跟一个空格和相对或绝对文件名）：
#
# @ AcrylicHostsGroup1.txt
# @ AcrylicHostsGroup2.txt
#
# 使用大量的模式或正则表达式可能会显著降低Acrylic的性能。从性能角度来看，最好使用
# 大量域名列表，而不是使用大量的模式和正则表达式。模式和正则表达式应该用于在域名列表
# 完全未知或变化过多的情况下，方便手动维护。域名比较便宜，而模式相对便宜，正则表达式则比较昂贵。
#
